# クラスメッセージボード

卒業記念撮影で、生徒一人ひとりのiPadに担当文字を大きく表示するWebアプリです。代表者が文字・順番・配色を管理し、Firebase Realtime Databaseを介して参加者の端末へ即時反映します。

## 主な機能

- 参加者：担当番号の選択・記憶、前後の文字確認、縦横対応の撮影表示、2.3秒長押しの簡易ロック解除
- 代表者：メッセージ分割、文字・順番・個別色編集、配色パターン、完成プレビュー、QRコード、JSONバックアップ
- 共通：Firebaseリアルタイム同期、最終設定の端末保存、通信切断時の表示維持、iPad Safari向け安全領域と文字サイズ自動調整
- Firebase未設定時：デモモードで全画面を確認可能

## ローカル起動

Node.js 22.13以降を用意し、プロジェクトのフォルダで次を実行します。

```text
npm install
npm run dev
```

表示されたURLをブラウザで開きます。参加者画面は `/?room=demo8k2m`、代表者画面は `/?room=demo8k2m&admin=1` です。

## Firebaseの準備

1. Firebase Consoleで新規プロジェクトを作ります。
2. 「プロジェクトの設定」からWebアプリを登録し、表示された設定値を控えます。
3. Realtime Databaseを作成します。学校での利用地域に近いロケーションを選びます。
4. Authenticationの「ログイン方法」で「メール／パスワード」を有効にします。
5. Authenticationの「ユーザー」で代表者アカウントを作成し、そのUIDを控えます。
6. `.env.example` を `.env.local` へコピーし、Firebase Webアプリの値を入力します。パスワードやサービスアカウント鍵は入力しません。
7. Realtime Databaseに `admins/{代表者UID}: true` を登録します。
8. `firebase/database.rules.json` の内容をRealtime Databaseの「ルール」へ貼り付けて公開します。
9. 開発サーバーを再起動します。代表者画面にログイン欄が表示されれば設定完了です。

FirebaseのWeb設定値は公開クライアント識別情報です。書き込み保護はSecurity RulesとAuthenticationで行います。代表者のパスワード、秘密鍵、サービスアカウントJSONはリポジトリへ保存しないでください。

## ルームと参加者URL

参加者URLは `https://公開URL/?room=ランダムなルームID` です。管理画面で編集したルームIDとURLの `room` を一致させます。QRコードは管理画面内で端末上に生成され、外部QR生成APIへURLを送りません。

初回は管理画面の「JSON保存」でバックアップを作成してから編集することを推奨します。「JSON復元」は形式を検査し、確認後に編集内容へ読み込みます。復元後は「変更を保存・反映」を押すまでFirebaseへ送信されません。

## iPadでの操作

1. 参加者用QRコードをSafariで開きます。
2. 自分の担当番号を選び、表示文字を確認して決定します。番号は端末に保存されます。
3. 「撮影表示にする」を押します。iPadは縦・横どちらでも利用できますが、撮影するカメラは横向きにします。
4. 撮影表示を終えるときは、右下の薄い「長押し」を約2.3秒押し続け、確認画面で終了します。

通信が切れても最後に受信した文字と色を表示し続けます。再読み込み時も端末内の最終設定を先に復元します。Safariを閉じる、ホームへ戻る、画面回転を固定するなどのiPad本体の操作はWebアプリから制限できません。

## Cloudflare Pages / Sitesへの公開

このプロジェクトはCloudflare Worker互換のビルドを出力します。

```text
npm run build
```

GitHubにこのフォルダを保存し、Cloudflare Pagesでリポジトリを接続します。Production branchは `main`、ビルドコマンドは `npm run build` を指定します。公開後はFirebase Authenticationの「承認済みドメイン」に発行されたドメインを追加してください。GitHub Pagesは使用しません。

## 当日の確認

- 学校のiPadとネットワークで公開URL、Firebase、QRコードを確認する
- 縦向き・横向き・端末回転後の1文字／2文字表示を確認する
- 代表者端末で文字と色を変え、別端末へ反映されることを確認する
- 未ログインの参加者がデータを書き換えられないことを確認する
- 最終設定のJSON、担当一覧のスクリーンショット、白い紙と太い黒ペンを用意する
- 写真は横向きで2〜3枚ずつ撮り、反射と文字切れを確認する

## テスト

```text
npm test
```

分割、配色優先順位、並び順、バックアップ検証に加え、Firebase Realtime Database Emulatorで以下を検証した後、公開用ビルドも確認します。

- `admins/{UID}: true` の管理者が、管理画面と同じルームデータを保存できる
- 未認証ユーザーおよび管理者以外の認証済みユーザーは保存できない
- 個別背景色・文字色が `null`（Realtime Database上では未保存）でも保存できる

ルールテストにはJava 21以上が必要です。`firebase.json` はテスト用のローカルEmulator設定であり、本番Firebaseの設定値や認証情報は含みません。
